Ruby varialbe scoping

Ruby was created with object oriented best practices in mind. As such, it has some built-in constraints that (while mysterious or possibly downright annoying to the noob) help Rubyists create maintainable code.

While tempting for the beginner Rubyist to create variables that are accessible anywhere in the program (global), or to create all instance variables as both readable and write-able regardless of their context or intended use, this would undoubtedly cause rampant unintended side-effects. Good practice dictates objects be encapsulated, which is to say only the most essential information is shared with other programmers, end-users and the rest of the program itself. Details of the specific inner workings are intentionally hidden as a safeguard against unintentional modifications.

Variable scope in Ruby starts out very tightly restricted, with local variables being only accessible within the method they were created. Once the method is done running, the variable disappears. Instance variables are available to the entire instance of a class. Class methods are also available across the entire scope of the class with a very important caveat: there is only ever one copy of a class variable at one time. As such, class variables should be used sparingly or not all. Global variables are extremely scary and best avoided. Their scope is infinite and can be altered anywhere within the program. Ruby comes with some built-in global variables that can be quite useful, but in general, these are more trouble than they worth.
