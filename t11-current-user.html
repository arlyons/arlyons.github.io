Rails App Part III

Linking the current user to our models.

In my last post we added the resources for our app. Let's keep building:

Revise our 'snippets_controller' methods:

Since our 'Snippet model' belongs to our 'User' model, we'll need to use build when defining our snippet instance variable in our 'snippets_controller' file.

  1. In our 'new' method, replace:

    2. @snippet = Snippet.new

    With the line:

    @snippet = current_user.snippets.build

  2. In our 'create' method, replace:

    3. @snippet = Snippet.new(snippet_params)

    With the line:

    @snippet = current_user.snippets.build(snippet_params)

Restrict 'Edit' and 'Destroy' actions to signed-in users

  1. Add the following code at the top of the 'snippet_controller' page:

    2. before_action :authenticate_user!

  2. In our 'snippets_index.html.erb' view file, add the following if statement inside our td tags at the bottom or our form:

    3. Replcace: <%= link_to 'Destroy', snippet, method: :delete, data: { confirm: 'Are you sure?' } %>

    With:

    1. <% if snippet.user == current_user %>
    2. <%= link_to t('.edit', :default => t("helpers.links.edit")), edit_snippet_path(snippet), :class => 'btn btn-default btn-xs' %>
    3. <%= link_to t('.destroy', :default => t("helpers.links.destroy")), snippet_path(snippet), :method => :delete, :data => { :confirm => t('.confirm', :default => t("helpers.links.confirm", :default => 'Are you sure?')) }, :class => 'btn btn-xs btn-danger' %>
    4. <% end %>

Now our "snippets" can only be created by a logged-in user, and only the owners of that snippet are able to edit and delete them. We're one crucial step closer to making our app CRUD.
