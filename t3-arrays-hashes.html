Arrays vs. Hashes

A look at the two most common data structures

Posted by Ali on 08/16/2015

Both arrays and hashes store values that can be accessed and manipulated by the user as well as other Ruby programs.

Arrays store these values via an index starting at 0.

Hashes are a bit more versatile in that they store key-value pairs, where the keys can be any Ruby object of your choosing.

Both have advantages and drawbacks, and one may be more suited than the other depending on your goals.

Arrays are ideal if you need to store values in an ordered list. Ruby will take the items in the array and index them starting at the number 0 (not 1!). The first item can be retrieved using the following: array_name[0]. If you don't know how large the index of an array, but you need to access the last item, Ruby allows you to access it via: array_name[-1]. There are a total of about 73 Ruby array methods, allowing for adding, deleting, traversing over and transforming the items in the array.

Hashes use key-value pairs wherein the key can be any object type you assign to it, i.e., a string: "name" => "Ali". Some examples of key, value pairs you might use a hash for are: states and their abbreviations, vocabulary words and their definitions or employees and their positions at a company company.
