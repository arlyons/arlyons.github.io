Regex in Ruby

Three common use cases demistyified.

Posted by Ali on 10/11/2015

As new programmers, there is one thing we can all agree on: Regual Expressions are scaaarrreee.... Actually, the notation is anything but pretty, but the basic syntax is pretty straightforward. Below I'll cover three very basic uses of Regular Expressions using Ruby.

Substitutions

Regular Expressions are ideal for making substitutions within strings. Let's say we have the following string: 

a = "doohickeses is the password"

but someone leaked the password and now we need to change it to something more secure: stickywicketisntcricket.

We'll use the #sub method to swap out the two passwords using regular expressions. We know that doohicikeses occupies the first 12 characters of our string. In order to pinpoint these first 12 characters using regular expressions, we will need three symbols:

  • an opening / and a closing / which enclose all of our regular expressions
  • the carrot A.K.A. anchor ^ (located above the 6 key), to indicate where to start the scan
  • dot(s) . which represent characters in our string

Here is what the code looks like: 

a = "doohickeses is the new password"

a.sub(/^............/, "stickywicketisntcricket ")

Extracting Numbers from Strings

Let's say you wanted to extract only the digits from a string.

  • once again you'll need \ to offset what you'll be searching for
  • and d for digtits
  • you'll also most likely need the + sign to signify we'll be looking for any number of joining digits, not just single digits

x = "What's the deal with the elevator skipping floor 13?"
x.scan(/\d+/) {|y| puts y}

Ouput: 

13

Let's say you have a website that requires users to enter their names and you want to throw an error message if they enter any numbers in their name. You'll need the following symbols:

Character Classes (Groups and Ranges)

You can use bracket notation to include ranges of numbers and letters to be scanned: 

c = "Now, why you would ever want to take out letters a through g out of a perfectly good sentence like this one?"
c.scan(/[a-g]/)

Output: 

["d", "e", "e", "a", "a", "e", "e", "e", "a", "g", "g", "f", "a", "e", "f", "e", "c", "g", "d", "e", "e", "c", "e", "e", "e"]

Above I used the =~ symbols, but you could also use the #match method: 

name = "Ali50"
puts "Name cannot contain numbers" if name.match(/[a-z]/)

Output: 

Name cannot contain numbers
